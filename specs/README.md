# Application Specifications

This directory is intended for **your own application specifications and documentation**. 

When building new features, tracking bugs, or collaborating with AI engineering assistants (like GitHub Copilot), it is highly recommended to partition your concepts into tiny Markdown files here (e.g., `01-auth-flow.md`, `02-database-schema.md`).

> *Note: The original documentation and architectural decisions regarding the underlying setup of this boilerplate (OpenNext + Pulumi) have been archived to `docs/boilerplate-reference/` to keep this folder clean for your app logic.*
